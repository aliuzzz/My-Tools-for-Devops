## CDN爬虫脚本 - v4-win 性能优化（15:10）

### 优化背景
用户反馈脚本运行慢（几十秒），经分析确认纯 requests 直连下载不可行，必须在走完整 UI 的前提下优化等待策略。

### 优化项（全部在 `cmcc_cdn_spider_v4-win.py` 上完成）
1. **OCR 移到循环外初始化**：`ddddocr.DdddOcr()` 只在登录前初始化一次，循环内只做 `classification()`
2. **内存截图不落盘**：`img_element.screenshot_as_png` 直接获取 bytes 传给 OCR，删除 `v_code.png` 落盘逻辑
3. **登录成功正向等待**：点击登录后正向等待"统计分析"菜单出现（最多3秒），替代原来的"等2秒看有没有错误提示"（稳定浪费2秒）
4. **验证码刷新等 src 变化**：记录旧验证码 src，点击后等 src 变化再识别，替代固定 `sleep(1)`
5. **域名下拉条件等待**：`sleep(5)` → `WebDriverWait` 等下拉列表 li 出现（最多3秒）
6. **checkbox 点击轮询**：`_click_checkbox_in_li` 完全重写，每次点击后轮询检查勾选状态（0.2秒间隔，最多2秒），替代原来的固定 `sleep(2)`
7. **各步骤 sleep 大幅缩短**：
   - 证书类型选择后：移除 `sleep(1)`
   - 昨日点击后：`sleep(3)` → `sleep(0.5)`
   - 确认点击后：`sleep(2)` → `sleep(0.3)`
   - 最小单位点击后：`sleep(1)` → `sleep(0.3)`
   - 导出后：`sleep(5)` → `sleep(2)`
   - 强制勾选后：`sleep(1)` → `sleep(0.3)`
   - body点击后：`sleep(1)` → `sleep(0.3)`

### 预估收益
固定等待从约 25-30 秒压缩到约 5-8 秒，整体执行时间预计从几十秒降到 10-15 秒（取决于网络和验证码一次成功率）。
