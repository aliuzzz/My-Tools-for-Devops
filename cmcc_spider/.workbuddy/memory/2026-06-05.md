## 2026-06-11 版本重构

### 版本结构调整
- **删除旧版本**：v3（Flask API）、v4（配置外置Flask）、v4.1（旧推送版）全部删除
- **重写 v4**：基于 v2（纯爬虫脚本）+ YAML 配置外置化
- **重写 v4.1**：基于新 v4 + XLS 解析 + 采样数据推送

### v2 版本
- 基础爬虫：登录 → 域名筛选 → 昨日 → 导出XLS（1000进制）
- 配置硬编码在脚本内
- 文件：`cmcc_cdn_spider_v2.py`

### v4 版本（基于v2重构）
- **核心改动**：用户名、密码、CP_ID、域名列表等全部提取到 YAML 配置文件
- **支持多客户**：一个 YAML 文件配置多个客户，通过 `--customer` 指定，或 `--all` 批量执行
- **用法**：
  - `python cmcc_cdn_spider_v4.py` — 默认跑第一个客户
  - `python cmcc_cdn_spider_v4.py --list` — 列出所有客户
  - `python cmcc_cdn_spider_v4.py --customer "淮北移动"` — 指定客户
  - `python cmcc_cdn_spider_v4.py --all` — 批量执行所有客户
- 保持 v2 的纯爬虫结构，不引入 Flask 等额外复杂度
- 文件：`cmcc_cdn_spider_v4.py`

### v4.1 版本（基于v4 + 推送接口）
- **功能**：下载XLS → 解析采样数据 → 推送到对方 `/system/cdn/traffic/samples` 接口
- **数据转换**：
  - XLS 第1列=时间（"HH:MM"），拼接日期 → `yyyy-MM-dd HH:mm:ss`
  - 第2列=带宽(bps)，转换 → Mbps（`bps / 1_000_000`）
- **推送配置**（每个客户的 `push_api` 段）：
  - url: 对方接口地址
  - customer_id: 客户ID
  - customer_name: 客户名称
  - headers: 额外请求头（可选）
- **用法**：
  - `python cmcc_cdn_spider_v4.1.py` — 默认跑第一个客户，抓取后自动推送
  - `python cmcc_cdn_spider_v4.1.py --all` — 批量执行所有客户
  - `python cmcc_cdn_spider_v4.1.py --skip-push` — 只下载不推送
- 文件：`cmcc_cdn_spider_v4.1.py`

### 配置文件结构（config.yaml）
```yaml
customers:
  - name: "安徽移动"
    user: '15366408811'
    passwd: 'xxx'
    cp_id: 'xxx'
    target_domains: [...]
    push_api: {...}
  - name: "淮北移动"
    user: '18970873839'
    ...
defaults:
  unit_scale: 1000
  download_dir: "."
```

### 配套文件
- `config.yaml` — 多客户配置文件（安徽移动 + 淮北移动）
- `requirements.txt` — 依赖清单
