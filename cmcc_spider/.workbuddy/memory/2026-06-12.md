## CDN爬虫脚本 - Linux版与Win版同步（13:40）

### 背景
用户要求Linux版v4和v4.1与Windows版保持功能一致。

### 同步内容
**`cmcc_cdn_spider_v4.py`（Linux纯下载版）：**
- 完全基于v4-win覆盖，保留Linux特有参数 `--no-sandbox`、`--disable-dev-shm-usage`
- 同步：`_create_driver`提取、文件名带客户名、按客户分目录(`xls/{客户名}/`)

**`cmcc_cdn_spider_v4.1.py`（Linux下载+推送版）：**
- `prefs['download.default_directory']` 改为 `os.path.abspath(download_dir)`
- 验证码刷新改为JS点击（绕过footer遮挡）
- 文件名带客户名、按客户分目录
- `parse_xls_to_samples` 加时间格式判断（兼容XLS已有完整日期的情况）
- `push_samples` 放宽成功判断（兼容code=0/200/success=true等多种返回）

### Linux与Win的核心差异
| 差异项 | Linux版 | Win版 |
|--------|---------|-------|
| Chrome参数 | 有 `--no-sandbox` `--disable-dev-shm-usage` | 无 |
| 默认下载目录 | 脚本所在目录 | `%USERPROFILE%\Downloads` |
| chromedriver路径 | 支持配置 | 支持配置 |

## CDN爬虫脚本 - 推送接口测试脚本（14:34）

### 背景
用户反馈每次测试推送接口都要跑完整爬虫流程太麻烦，需要一个独立的快速测试工具。

### 新增文件
- `test_push_api.py` — 推送接口连通性测试脚本

### 用法
```bash
python test_push_api.py                    # 测试第一个客户
python test_push_api.py --customer "淮北移动"  # 测试指定客户
```

### 功能
- 读取 `config.yaml` 中客户的 `push_api` 配置
- 构造1条mock采样数据（`trafficValue=123.456 Mbps`）
- 发送POST请求，打印完整请求体和响应
- 对HTTP 401给出明确提示（认证失败，检查headers配置）
- 对ConnectionError给出排查建议（服务是否启动、IP端口是否正确）
